# 📚 データカタログ（練習用データセット一覧）

この講座では、メイン教材の架空EC「Sora Mart」に加えて、**データの“形”や分野が異なる発展用データセット**を用意しています。
実務のデータは1種類ではありません。時系列・比率・アンケート・センサー・金融…と、**形が変われば見るべき指標も手法も変わります**。
いろいろなパターンに触れて、「このデータならまず何を見るか」の引き出しを増やしましょう。

> 💡 発展用データは `python scripts/generate_extra_datasets.py` で生成されます（乱数シード固定で再現可能）。
> メインのECデータは `python scripts/generate_data.py` です。

---

| データセット | データの形（パターン） | 主に練習できること | 関連モジュール |
|---|---|---|---|
| **Sora Mart EC**（メイン） | 正規化された関係データ（複数テーブル） | 結合・集計・KPI設計・一気通貫分析 | 02〜09 |

## 2. 📈 web_traffic_daily — 日次Webアクセス（時系列パターン）

`data/timeseries/web_traffic_daily.csv` … **731行**（2023-01-01〜2024-12-31）

| 列 | 説明 |
|----|------|
| `date` | 日付 |
| `sessions` | セッション数 |
| `users` | ユーザー数 |
| `pageviews` | ページビュー数 |
| `bounce_rate` | 直帰率（0〜1） |
| `conversions` | コンバージョン数 |
| `revenue` | 売上（円） |

**仕込まれた特徴**：右肩上がりのトレンド／週次の季節性（**週末はアクセスが落ちる**）／キャンペーンによる数回の急増（スパイク）／**8日分の欠測（NaN）**。

🖊 **練習できる問い**
- 月次・週次に集計して推移を折れ線で可視化する
- 7日移動平均でノイズをならし、トレンドを見る
- 曜日別の平均セッションを比較する（週末効果は？）
- 欠測日をどう扱うか（前後の平均で補間 vs 除外）を判断する
- 平常時から大きく外れた「異常な日」を検出する（例：平均±3σ）

## 3. 🎯 ad_campaigns — 広告チャネル別実績（比率・ファネルパターン）

`data/marketing/ad_campaigns.csv` … **910行**（5チャネル × 約半年の日次）

| 列 | 説明 |
|----|------|
| `date` | 日付 |
| `channel` | チャネル（検索広告／ディスプレイ／SNS広告／動画広告／アフィリエイト） |
| `impressions` | 表示回数 |
| `clicks` | クリック数 |
| `cost` | 広告費（円） |
| `conversions` | コンバージョン数 |
| `revenue` | 売上（円） |

**ポイント**：`impressions → clicks → conversions` というファネル構造。チャネルごとに効率が違うので**比較**が主役。

💡 **覚えておきたい広告指標**
| 指標 | 意味 | 計算 |
|------|------|------|
| CTR | クリック率 | clicks / impressions |
| CVR | コンバージョン率 | conversions / clicks |
| CPC | クリック単価 | cost / clicks |
| CPA | 獲得単価 | cost / conversions |
| ROAS | 広告費用対効果 | revenue / cost |

🖊 **練習できる問い**
- チャネル別に CTR / CVR / CPA / ROAS を計算し、ランキングする
- 「クリックは多いがCVが少ない」チャネルを見つける（ファネルのどこで落ちている？）
- ROAS が最も高いチャネルはどこか。予算を移すならどこからどこへ？
- 日次のコストと売上の推移を可視化し、効率が悪化した時期を探す

## 4. 🗳 customer_survey — 顧客アンケート（カテゴリ・リッカートパターン）

`data/survey/customer_survey.csv` … **1,200行**（1人1回答）

| 列 | 説明 |
|----|------|
| `respondent_id` | 回答者ID |
| `age_group` | 年代（10代〜60代以上） |
| `gender` | 性別 |
| `region` | 地域ブロック |
| `membership` | 会員種別（無料／有料） |
| `satisfaction` | 総合満足度（**1〜5のリッカート尺度**） |
| `q_quality` / `q_price` / `q_support` | 品質・価格・サポートの評価（1〜5） |
| `recommend_score` | 推奨度（**0〜10：NPS算出用**） |
| `heard_from` | 知ったきっかけ |
| `free_comment` | 自由記述（短い定型コメント） |

**仕込まれた特徴**：**未回答（欠損）**が複数列にある／有料会員ほど満足度が高い構造／各項目評価と総合満足度が連動。

💡 **NPS（ネット・プロモーター・スコア）の出し方**
推奨度 9-10 を「推奨者」、0-6 を「批判者」とし、`NPS = 推奨者の割合 − 批判者の割合`（%）。

🖊 **練習できる問い**
- 会員種別・年代別に満足度の平均を比較する（クロス集計）
- NPS を全体・セグメント別に計算する
- 総合満足度と最も相関が強い項目はどれか（満足度の“ドライバー”分析）
- 自由記述コメントを集計し、ポジ/ネガの傾向をつかむ
- 未回答をどう扱うか（除外 vs 補完）を考える

## 5. 👥 employees — 従業員データ（混合型・回帰/群比較パターン）

`data/hr/employees.csv` … **1,500行**（1人1行）

| 列 | 説明 |
|----|------|
| `employee_id` | 従業員ID |
| `department` | 部署 |
| `job_level` | 職級（1〜5） |
| `age` | 年齢 |
| `tenure_years` | 勤続年数 |
| `education` | 学歴 |
| `gender` | 性別 |
| `work_location` | 勤務地 |
| `monthly_salary` | 月給（円） |
| `overtime_hours` | 月の残業時間 |
| `engagement_score` | エンゲージメント（1〜5） |
| `attrition` | 離職フラグ（1=退職, 0=在籍） |

**仕込まれた構造**：給与は**職級・勤続・学歴**で説明できる（→ 回帰の好例）。離職は**残業が多い・エンゲージメントが低い・給与が低い**ほど起きやすい（→ 要因分析）。

🖊 **練習できる問い**
- 部署別・職級別に平均給与を比較する
- 給与を職級・勤続・学歴で説明する重回帰を作り、係数を解釈する
- 離職した人としていない人で、残業時間やエンゲージメントに差があるか（群比較＋検定）
- どの要因が離職と最も関係しているか
- 男女で給与差があるか（他の条件をそろえて考える＝シンプソンのパラドックスに注意）

## 6. 🏭 sensor_readings — 製造ラインのセンサー（高頻度時系列・異常検知パターン）

`data/iot/sensor_readings.csv` … **1,344行**（4台 × 約2週間の毎時データ）

| 列 | 説明 |
|----|------|
| `timestamp` | 計測日時（1時間ごと） |
| `machine_id` | 機械ID（M-01〜M-04） |
| `temperature` | 温度（℃） |
| `vibration` | 振動 |
| `pressure` | 圧力 |
| `rpm` | 回転数 |
| `defect` | 不良発生フラグ（1=不良） |

**仕込まれた特徴**：温度の**緩やかなドリフト（劣化）**と日内変動／**突発的な外れ値（スパイク）**／**センサー欠損（NaN）**／不良は**高温×高振動**で発生しやすい。

🖊 **練習できる問い**
- 機械ごとに温度・振動の時系列を可視化し、劣化やスパイクを見つける
- 欠損値を時系列補間（前方補完・線形補間）で埋める
- IQR法や±3σで外れ値を検出する
- 不良が出たときと出ないときで、温度・振動に差があるか
- 不良を予測する簡単なルール（しきい値）を考える

## 7. 💹 stock_prices — 株価（複数時系列・相関パターン）

`data/finance/stock_prices.csv` … **2,500行**（架空5銘柄 × 500営業日, ロング形式）

| 列 | 説明 |
|----|------|
| `date` | 日付（平日のみ） |
| `ticker` | 銘柄名（架空5社） |
| `close` | 終値（円） |

**ポイント**：**ロング形式**（1行＝1銘柄1日）なので、`pivot` でワイド形式（日付×銘柄）に変換する練習になる。市場全体の動きで銘柄間に**相関**がある。

🖊 **練習できる問い**
- `pivot` でワイド形式に直し、各銘柄の終値を1枚の折れ線で比較する
- 日次リターン（前日比）を計算する（`pct_change`）
- 銘柄ごとのリターンの平均・標準偏差（＝リスク）を比較する
- 銘柄間のリターン相関行列を作り、ヒートマップで可視化する
- 累積リターン（1から始めた指数）を計算して比較する
