## 6-1. なぜ可視化が要るのか

表の数字は「正確」ですが、人間は数百行の表から傾向を読めません。
可視化は **「形」にして直感に渡す**作業です。上がっている／偏っている／外れている、を一瞬で伝えます。

ただし可視化は諸刃の剣でもあります。**見せ方しだいで誤解させられる**（後述の軸の切り詰め）。
だからこそ「正しく・伝わるように」描く技術が要ります。

> 本講座の分析規約のおさらい：**売上 = `quantity × unit_price` の合計、`status == 'completed'` のみ**。
> グラフでも必ずこの定義で集計します。

## 6-2. matplotlib の基本 ── figure と axes の2階建て

matplotlib の図は2階建てです。

- **Figure**：キャンバス全体（1枚の紙）
- **Axes（ax）**：その上の1つのグラフ領域。線・軸・凡例はここに描く

```python
fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(x, y, marker="o", label="2024年")
ax.set_title("月別の売上")
ax.set_xlabel("月")
ax.set_ylabel("売上（万円）")   # ← 単位まで書く
ax.legend()
plt.show()   # Jupyterではここで表示。スクリプト(Agg)では表示されないだけ
```

どんなグラフにも最低限つけるもの：

| 要素 | メソッド | なぜ必要か |
|------|----------|-----------|
| タイトル | `ax.set_title()` | 何のグラフか一言で |
| 軸ラベル | `ax.set_xlabel()` / `ax.set_ylabel()` | **単位**まで（円・人・%） |
| 凡例 | `ax.legend()` | 複数系列があるとき何が何か |

> 💡 **日本語フォント**を設定しないと文字が「□（豆腐）」になります。
> セットアップで Meiryo / Yu Gothic などを `rcParams["font.family"]` に設定済みです。

## 6-3. seaborn で簡潔に描く

**seaborn** は matplotlib の上に乗ったライブラリ。DataFrame と列名を渡すだけで集計＋見栄えよく描けます。

```python
sns.barplot(data=df, x="category", y="amount", ax=ax)   # 棒
sns.histplot(data=df, x="age", bins=20)                 # ヒストグラム
sns.boxplot(data=df, x="category", y="amount")          # 箱ひげ
sns.scatterplot(data=df, x="age", y="amount")           # 散布図
sns.heatmap(df.corr(), annot=True)                      # ヒートマップ
```

使い分けの目安：**素早く探索＝seaborn、作り込み＝matplotlib**。
本講座では「seaborn で描いて、返ってきた `ax` を matplotlib で仕上げる」が基本形です。

## 6-4. グラフの選び方 ── 目的→グラフ種の早見表

「何を伝えたいか」が決まれば、グラフはほぼ自動的に決まります。**これだけは暗記**してください。

| 伝えたいこと（目的） | 向いているグラフ | 例（Sora Mart） |
|--------------------|-----------------|------------------|
| 時間に沿った**推移** | 折れ線 | 月次売上の推移 |
| カテゴリ間の**比較** | 棒（横棒も可） | カテゴリ別売上 |
| 1変数の**分布** | ヒストグラム / 箱ひげ図 | 年齢、注文金額 |
| 2変数の**関係** | 散布図 | 年齢 vs 累計購入額 |
| **構成比**（内訳の比較） | 棒（積み上げ） | 経路×カテゴリ |
| 多変数の**相関の俯瞰** | ヒートマップ | 顧客指標の相関 |

> 🧭 迷ったら4語：**推移=線 / 比較=棒 / 分布=ヒスト・箱 / 関係=散布図**。

### 円グラフの注意点

構成比＝円グラフ、と反射的に選びがちですが、円グラフは弱点が多いグラフです。

- 人間は**角度の比較が苦手**（長さの比較は得意）
- 項目が多いと読めない／並べ替えで順位が伝わりにくい

→ 多くの場合、**降順の棒グラフ**や**積み上げ棒**の方が正確に伝わります。
円グラフを使うなら「**項目2〜3個**」「合計が100%だと強調したい」ときに限定しましょう。

## 6-5. 「伝わる」グラフの原則

同じデータでも、見せ方で伝わり方は激変します。実務で効く5原則：

1. **1グラフ1メッセージ**：そのグラフで言いたいことを1つに絞る。欲張らない。
2. **軸ラベルと単位**：「売上」ではなく「売上（円）」。読み手に推測させない。
3. **棒グラフはソートする**：カテゴリ比較は**大きい順**に。順位が一瞬で分かる。
   （月・曜日など「順序に意味がある」軸はソートしない。）
4. **装飾を減らす**：3D・影・過剰な色は情報を埋もれさせる。インクは情報のために。
5. **誤解を生む表現を避ける**：棒グラフの**y軸は0から**。途中で切ると差が誇張される。

### ❌ 悪い例 / ⭕ 良い例 ── y軸の切り詰め

3店舗の売上が `100, 102, 104`（ほぼ横並び）のとき：

| | y軸の起点 | 見え方 | 評価 |
|--|----------|--------|------|
| ❌ 悪い | 99 から | わずかな差が「大差」に見える | 誤認させる |
| ⭕ 良い | 0 から | ほぼ横並びだと正しく伝わる | 正直 |

ニュースやプレゼンで「差が大きく見える棒グラフ」を見たら、まず**y軸の起点**を疑いましょう。

### 良い例／悪い例まとめ

| 悪い例 | なぜダメか | 良い例 |
|--------|-----------|--------|
| 軸ラベルなし | 何の数字か不明 | 「売上（円）」と単位つき |
| 棒が順不同 | 順位が読めない | 降順にソート |
| 3D円グラフで内訳 | 角度＋奥行きで歪む | 降順の棒 or 積み上げ棒 |
| 全系列カラフル | どこを見るか不明 | 主役だけ色、他はグレー |
| y軸を途中から | 差を誇張 | y軸は0から |

## 6-6. 複数グラフを並べる（subplots）

`plt.subplots(行, 列)` で複数の axes を一度に作り、`axes[行, 列]` に描き分けます。
関連する図を1枚にまとめると「全体像」を伝える**ダッシュボード**になります。

```python
fig, axes = plt.subplots(2, 2, figsize=(13, 9))
axes[0, 0].plot(...)        # 推移
sns.barplot(..., ax=axes[0, 1])   # 比較
sns.histplot(..., ax=axes[1, 0])  # 分布
sns.scatterplot(..., ax=axes[1, 1])  # 関係
fig.suptitle("ミニダッシュボード")
plt.tight_layout()   # 図の重なりを防ぐ
```

## 6-7. ビジネスでの見せ方 ── 主役を立てる

意思決定の場では「**どこを見ればいいか**」を作り手が指し示します。

- **強調したい1本だけ色を変え、残りはグレー**にする → 視線が一瞬でそこへ。
- `ax.annotate("ここが言いたい", xy=..., xytext=..., arrowprops=...)` で図に直接注釈を置く。

```python
colors = ["#B0B0B0"] * n   # 全部グレー
colors[0] = "#C44E52"      # 主役だけ赤
ax.bar(labels, values, color=colors)
ax.annotate("最大", xy=(0, top), xytext=(0.8, top*0.9),
            arrowprops=dict(arrowstyle="->"))
```

> 💡 「グラフを見せる」のではなく「**結論を見せて、グラフで裏づける**」。
> モジュール01の「結論ファースト」と同じ精神です。
