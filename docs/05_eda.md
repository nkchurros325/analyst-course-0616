# 05. 記述統計とEDA（探索的データ分析）

> このモジュールのゴール：**データを触ったら、まず「分布・代表値・ばらつき・外れ値」を見て素性をつかむ**ことを、手と頭の両方で身につける。
> 平均だけで語らず、「で、何が言えるか」を一言で書く習慣をつけます。

前提：モジュール02〜04（SQLとpandasでのデータ取得・前処理）を終えていること。コードは pandas / numpy / matplotlib / seaborn を使います。

---

## 5-1. EDA とは何か、なぜ最初にやるのか

EDA（Exploratory Data Analysis：探索的データ分析）は、**いきなりモデルや結論に飛びつく前に、データの形・分布・関係をざっくり把握する**作業です。

- 数字を1つ（平均など）に潰す前に、**全体の形**を見る。
- 想定外の値（外れ値・欠損・入力ミス）を**早期に発見**する。
- 「次にどんな分析をすべきか」の**仮説を立てる**材料を集める。

> 💡 EDAは「答えを出す」段階ではなく「問いを研ぐ」段階です。ここを飛ばすと、汚れたデータや偏った分布の上に分析を積み上げてしまい、あとで全部やり直しになります。

---

## 5-2. 記述統計量 ── データを少数の数字で要約する

データの素性は、いくつかの**代表値（中心）**と**ばらつき（散らばり）**で要約できます。

| 種類 | 統計量 | 意味 | pandas |
|------|--------|------|--------|
| 中心 | 平均（mean） | 合計 ÷ 個数。外れ値に弱い | `s.mean()` |
| 中心 | 中央値（median） | 真ん中の値。外れ値に強い | `s.median()` |
| 中心 | 最頻値（mode） | 最も多く出る値。カテゴリ向き | `s.mode()` |
| 散らばり | 分散（variance） | 平均からのズレの2乗の平均 | `s.var()` |
| 散らばり | 標準偏差（std） | 分散の平方根。元の単位に戻る | `s.std()` |
| 位置 | 四分位数（quartile） | 25/50/75%点。Q1, Q2(中央値), Q3 | `s.quantile([.25,.5,.75])` |
| 位置 | パーセンタイル | 下から何%の位置か | `s.quantile(0.9)` |
| 幅 | 範囲（range） | 最大 − 最小 | `s.max()-s.min()` |

これらを一気に出すのが **`describe()`** です。

```python
df["age"].describe()
# count  件数（欠損を除く）
# mean   平均
# std    標準偏差
# min    最小
# 25%    第1四分位数 Q1
# 50%    中央値 Q2
# 75%    第3四分位数 Q3
# max    最大
```

> 💡 **標準偏差の読み方**：分布がおおむね釣鐘型なら、「平均 ± 1標準偏差」の範囲におよそ7割弱が入る、というのが目安です。std が大きいほど人によるばらつきが大きい、と読みます。

## 5-3. 平均 vs 中央値 ── どちらで語るか

平均（mean）は計算が簡単で広く使われますが、**外れ値に引っ張られる**弱点があります。

例：年収 400, 420, 450, 480, **9000**（万円）の5人。
- 平均 = 2,150 万円（誰の実感とも合わない）
- 中央値 = 450 万円（真ん中の人の感覚に近い）

```
分布が左右対称  → 平均 ≒ 中央値
右に裾が長い    → 平均 > 中央値（高額の外れ値が平均を持ち上げる）
左に裾が長い    → 平均 < 中央値
```

売上・購入額・年収など「下限はあるが上限が青天井」のデータは**右に歪む（右裾が長い）**ことが多く、平均だけ見ると実態より大きく見えます。

> 💡 使い分けの目安：**「典型的な人/注文はどれくらい？」を語るなら中央値**、**「全体の合計や1件あたりの平均負担」を語るなら平均**。両方出して差を見ると、分布の歪みに気づけます。

## 5-4. 分布の形を見る ── ヒストグラム・歪度・尖度

代表値だけでは「形」は分かりません。**ヒストグラム**で分布を目で見ましょう。

```python
import matplotlib.pyplot as plt
df["age"].plot.hist(bins=30)
plt.show()
```

- **歪度（skewness, `s.skew()`）**：分布の左右非対称さ。
  - `> 0`：右に裾が長い（右歪み）。平均 > 中央値。
  - `≒ 0`：ほぼ対称。
  - `< 0`：左に裾が長い（左歪み）。
- **尖度（kurtosis, `s.kurt()`）**：分布の尖り具合・裾の重さ。正規分布を0とする定義（pandas/scipyの既定）で、`> 0` は裾が重く外れ値が出やすい、`< 0` は平べったい。

> 💡 尖度は中〜上級トピックです。本講座では「裾の重さの目安」くらいに留め、まずは**歪度の符号とヒストグラムの形**を読めれば十分です。

## 5-5. カテゴリ変数の要約 ── value_counts と構成比

数値ではないカテゴリ（性別・カテゴリ・獲得経路など）は、平均ではなく**度数（件数）と構成比（割合）**で要約します。

```python
df["category"].value_counts()                  # 件数の多い順
df["category"].value_counts(normalize=True)    # 構成比（合計1.0）
```

> 💡 構成比は「全体の何割を占めるか」を見るのに必須。件数だけだと母数が違うグループ同士を比べられません。「比較するなら割合に直す」が鉄則です。

---

## 5-6. 外れ値の検出 ── IQR法と箱ひげ図

外れ値（outlier）は、入力ミスのこともあれば、**本物の重要な少数（VIP顧客・高額注文）**のこともあります。まず**検出して中身を確かめる**のが基本で、機械的に消すのは厳禁です。

代表的な検出法が **IQR法**です。四分位範囲 IQR = Q3 − Q1 を使い、

```
下側の境界 = Q1 − 1.5 × IQR
上側の境界 = Q3 + 1.5 × IQR
この範囲の外を「外れ値の候補」とする
```

```python
q1, q3 = df["amount"].quantile([0.25, 0.75])
iqr = q3 - q1
upper = q3 + 1.5 * iqr
outliers = df[df["amount"] > upper]
```

**箱ひげ図（boxplot）**はこのIQRを図にしたものです。箱がQ1〜Q3、箱の中の線が中央値、ひげが境界、点が外れ値候補。

> 💡 外れ値を見つけたら「なぜこの値？」を必ず確認。**異常なら直す/除く、正常なら活かす**（高額注文＝大事な顧客かもしれない）。判断を飛ばして削除しないこと。

## 5-7. 2変数の関係を見る ── crosstab・groupby・corr

EDAの後半は「変数どうしの関係」です。**比較**こそ分析の本体です。

- **クロス集計（`pd.crosstab` / `pivot_table`）**：2つのカテゴリの組み合わせごとの件数や平均。
  ```python
  pd.crosstab(df["acquisition_channel"], df["gender"])
  df.pivot_table(index="category", values="amount", aggfunc="sum")
  ```
- **グループ別要約（`groupby`）**：セグメントごとに代表値を出す。
  ```python
  df.groupby("acquisition_channel")["amount"].agg(["count", "mean", "median"])
  ```
- **相関行列（`corr`）**：数値変数どうしの直線的な関係の強さ（−1〜1）。
  ```python
  df[["age", "n_orders", "total_amount"]].corr()
  ```

> 💡 相関は**直線的な関係の強さ**であって、**因果ではありません**。「相関が高い＝原因」ではないこと、相関が0でも非直線の関係はあり得ることに注意（モジュール01の「相関と因果の混同」も復習）。

## 5-8. 分析用データセットを組み立てる

実務のEDAは、生テーブルのままでは進みません。複数テーブルを **merge** して「**1行＝1注文**」や「**1行＝1顧客**」の分析用テーブルを作ります。

本講座の鉄則どおり、**売上は `quantity × unit_price` を合計し、`status == 'completed'` のみ**で集計します。

```python
# 明細に「金額」を作る
items["amount"] = items["quantity"] * items["unit_price"]

# 注文ヘッダ（completed のみ）と結合 → 注文ごとに金額を合計
oi = items.merge(orders[orders["status"] == "completed"], on="order_id")
order_amount = oi.groupby("order_id")["amount"].sum()   # 1行=1注文

# 顧客ごとの累計購入額・注文回数
per_order = oi.groupby(["customer_id", "order_id"])["amount"].sum().reset_index()
per_customer = per_order.groupby("customer_id")["amount"].agg(
    total_amount="sum", n_orders="count"
)
```

ここから **客単価（AOV）= 売上 ÷ 注文数**、**顧客別累計額**、**購入頻度**などの指標が計算できます（モジュール01の指標表を参照）。

> 💡 「分析用データセットを一度きれいに作る」と、その後の集計・可視化が一気に楽になります。**集計の単位（注文単位か顧客単位か）を最初に決める**のがコツです。

## 5-9. EDAの心得（まとめ）

1. **まず分布を見る**。平均を出す前にヒストグラムと `describe()`。
2. **平均だけで語らない**。中央値・ばらつき・外れ値とセットで。
3. **必ず比較する**。前年比・グループ間・セグメント間。単独の数字は解釈できない。
4. **仮説を持って見る**。「上位顧客に売上が偏っているのでは？」など問いを持つと、見るべき切り口が決まる。
5. **外れ値は確認してから扱う**。消す前に中身を見る。
6. **相関≠因果**。関係が見えても原因と決めつけない。

軽い可視化（ヒストグラム・箱ひげ図・棒グラフ）は素性把握のために使います。**見せるための本格的な可視化（配色・注釈・ストーリー設計）はモジュール06**で扱います。ここでは「自分が理解するための図」で十分です。
