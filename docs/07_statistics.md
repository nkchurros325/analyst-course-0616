# 07. 統計的推論と A/B テスト

> このモジュールのゴール：**手元の標本（サンプル）から母集団について何が言えるかを、信頼区間と仮説検定で語れるようになる。そして A/B テストを「正しく」読み解く。**
> 題材は `data/ab_test.csv`（`user_id, group(A/B), converted(0/1), session_seconds`）。group A はコントロール、group B はテスト施策、`converted` は購入の有無です。

---

## 7-1. なぜ「推論」が必要なのか

A/B テストでこんな結果が出たとします。

| group | 人数 | 購入数 | コンバージョン率(CVR) |
|-------|------|--------|----------------------|
| A（現行） | 6,200 | 274 | 4.42% |
| B（新施策） | 6,150 | 368 | 5.98% |

B のほうが高い。では「B は A より良い」と言い切ってよいでしょうか？

ここで立ち止まれるかどうかがアナリストの分かれ道です。私たちが見ているのは **たまたま集まった一部の利用者（標本）** であって、本当に知りたいのは **全利用者・将来の利用者まで含めた真の値（母集団）** です。標本は引くたびに少しずつ違う数字になります。だから問うべきは：

> **「この 4.42% → 5.98% という差は、偶然のブレで説明できる範囲か？ それとも本当に施策が効いたのか？」**

この問いに数字で答える道具が **統計的推論** です。

```
母集団（知りたい真の値）
   │  ……全部は観測できない
   ▼
標本（手元のデータ）  ──→  推論 ──→ 母集団について「幅つき」で語る
```

> 💡 推論の心構え：標本から得た数字には必ず誤差がある。**点（4.42%）ではなく幅（区間）で考える**のが第一歩です。

---

## 7-2. 確率分布のさわり ── 正規分布と中心極限定理

### 正規分布

左右対称の釣鐘型。平均 μ のまわりに、標準偏差 σ を物差しにデータが散らばります。目安として **約68% が ±1σ、約95% が ±2σ（厳密には±1.96σ）** の中に入ります。後で出てくる「95%信頼区間」の 1.96 はここから来ます。

### 中心極限定理（CLT）

統計的推論を支える最重要の事実です。

> 元のデータがどんな分布でも、**標本平均**を何度も計算してその分布を見ると、サンプルサイズが大きいほど **正規分布に近づく**。しかもその散らばり（標準誤差）は サンプルサイズが増えるほど小さくなる。

たとえば「0〜1の一様分布（まったく釣鐘型でない）」から 30 個取って平均する、を何千回も繰り返すと、平均値のヒストグラムはきれいな釣鐘型になります（ノートブックでシミュレーションします）。

この定理のおかげで、元データが正規分布でなくても「標本平均の振る舞いは正規分布で近似できる」と言え、信頼区間や検定の計算が成り立ちます。

> 💡 CLT が効くのは **平均（や割合）** の話。個々のデータ自体が正規分布になるわけではありません。

## 7-3. 標準誤差と信頼区間

### 標準誤差（SE）

**標準偏差（データのばらつき）** と **標準誤差（標本平均のばらつき）** は別物です。

```
標準誤差 SE = 標準偏差 s / √n
```

n（サンプルサイズ）が大きいほど SE は小さい＝平均の推定が安定する、を式が表しています。

### 平均の 95% 信頼区間

```
95%信頼区間 = 標本平均 ± 1.96 × SE
```

「95%信頼区間」の正しい意味は少しデリケートです。

- ⭕ 正しい：**同じやり方で標本を取り直して区間を作る、を100回やれば、約95回はその区間が真の母平均を含む**。
- ❌ よくある誤解：「真の平均がこの区間に入る確率が95%」。真の平均は定数なので「確率」で語るのは厳密には不正確。

実務では「真の値はだいたいこの範囲だろう」という幅として使えれば十分です。**区間が狭いほど推定の精度が高い**、と読みます。

### 割合（CVR）の信頼区間

割合 p̂ の標準誤差は次の式です（正規近似）。

```
SE = √( p̂(1−p̂) / n )
95%信頼区間 = p̂ ± 1.96 × SE
```

A 群の CVR 4.42%（n=6,200）なら SE ≈ 0.26%、信頼区間は概ね **3.9%〜4.9%**。B 群は **5.4%〜6.6%**。区間が重なっていないことが、後の検定結果（有意差あり）と整合します。

> 💡 グラフに平均だけ出して棒を並べるのは危険。**エラーバー（信頼区間）を添える**と「その差は誤差の範囲かどうか」が一目で伝わります。

## 7-4. 仮説検定の枠組み

「差は偶然か、本物か」を判定する標準手順です。

### ステップ

1. **仮説を立てる**
   - 帰無仮説 H₀：「差はない（B と A で CVR は等しい）」── まずこれを仮に正しいとする。
   - 対立仮説 H₁：「差がある（B と A で CVR は異なる）」── 示したいこと。
2. **有意水準 α を決める**（慣習で 0.05）。「これより珍しければ偶然とは考えにくい」という基準線。
3. **検定統計量を計算**（t値、カイ二乗値など）。
4. **p値を出す**。
5. **判断**：p < α なら H₀ を棄却（＝有意差あり）、p ≥ α なら棄却できない。

```
H₀（差はない）を仮定 → データはどれくらい珍しい？ → p値 → α=0.05 と比較 → 結論
```

### p値の「正しい」意味

> p値 = **「H₀（差がない）が本当だとしたら、いま観測されたかそれ以上に極端な差が、偶然で起きる確率」**。

- ❌ 「p値は H₀ が正しい確率」ではない。
- ❌ 「p値が小さい＝効果が大きい」ではない（効果の大きさは別途、後述の効果量で測る）。
- p が小さい＝「偶然でこんな差が出るのは考えにくい」→ 差は本物らしい、と読む。

### 2種類の誤り

| | H₀ が本当は正しい | H₀ が本当は誤り |
|--|------------------|----------------|
| **H₀ を棄却した** | 第一種の誤り（α）❌ 差がないのに「ある」と言う | ⭕ 正しい |
| **棄却しなかった** | ⭕ 正しい | 第二種の誤り（β）❌ 差があるのに「ない」と言う |

第一種＝**あわてんぼうの誤り**（効果がないものを採用してしまう）、第二種＝**ぼんやりの誤り**（効果があるのに見逃す）。α を下げると第一種は減るが第二種が増える、というトレードオフがあります。第二種を防ぐには十分なサンプルサイズ（検出力）が必要です。

## 7-5. 2群の比較 ── 実践

### (a) 連続値：session_seconds を t 検定

A と B の平均滞在時間に差があるか。独立した2群の平均比較なので **独立2標本 t 検定**（`scipy.stats.ttest_ind`）を使います。

```python
from scipy import stats
a = df[df["group"] == "A"]["session_seconds"]
b = df[df["group"] == "B"]["session_seconds"]
t, p = stats.ttest_ind(a, b, equal_var=False)  # Welchのt検定（分散が等しいと仮定しない）
```

- 帰無仮説 H₀：A と B の母平均は等しい。
- 結果：A 平均 ≈ 94.8 秒、B 平均 ≈ 100.5 秒、t ≈ −7.8、**p ≈ 7×10⁻¹⁵**。
- 結論：有意水準5%で H₀ を棄却。**滞在時間には統計的に有意な差がある**（B が長い）。

> 💡 `equal_var=False`（Welch）を既定にしておくと、2群の分散が違っても妥当。迷ったら Welch が安全。

### (b) 割合：converted のカイ二乗検定

CVR（割合）の差は、クロス集計表（group × converted）に対する **カイ二乗検定**（`scipy.stats.chi2_contingency`）で見ます。

```python
ct = pd.crosstab(df["group"], df["converted"])
chi2, p, dof, expected = stats.chi2_contingency(ct)
```

- 帰無仮説 H₀：group と converted は独立（＝CVR は群によらず同じ）。
- 結果：χ² ≈ 15.0、自由度 1、**p ≈ 0.0001**。
- 結論：有意水準5%で H₀ を棄却。**CVR には統計的に有意な差がある**（B が高い）。

#### もう一つの考え方：2標本の割合の z 検定

割合の差は z 検定でも検定できます（2×2 のカイ二乗とほぼ等価で、χ² = z²）。`scipy` だけで書くなら：

```python
import numpy as np
from scipy import stats
n_a, x_a = 6200, 274   # A群の人数・購入数
n_b, x_b = 6150, 368
p_a, p_b = x_a/n_a, x_b/n_b
p_pool = (x_a + x_b) / (n_a + n_b)               # H₀ の下での共通割合
se = np.sqrt(p_pool*(1-p_pool)*(1/n_a + 1/n_b))  # H₀ の下での標準誤差
z = (p_b - p_a) / se
p_value = 2 * (1 - stats.norm.cdf(abs(z)))       # 両側
```

z ≈ 3.87、p ≈ 0.0001 となり、カイ二乗とほぼ同じ結論に到達します。**「割合の差を見るときはカイ二乗 or z 検定」**と覚えておけば実務はまわります。

## 7-6. 効果量と実務的有意性

検定が教えるのは「差が偶然か否か」だけで、**差の大きさ（=ビジネスインパクト）** は別問題です。サンプルが巨大なら、ごくわずかな差でも p は小さくなります。

> **統計的に有意 ≠ ビジネス的に重要。**

CVR の差を実務語に翻訳しましょう。

- **絶対差**：5.98% − 4.42% = **+1.56 ポイント**
- **リフト（相対改善率）**：(5.98 − 4.42) / 4.42 = **約 +35%**

「CVR が相対35%改善」はインパクト大。さらに「月間訪問が10万なら購入が約1,560件増える」のように **事業の数字に換算** すると意思決定者に刺さります。連続値では Cohen's d などの効果量指標もありますが、まずは「絶対差・相対差・売上換算」で十分です。

> 💡 報告は必ずワンセット：**「有意か（p値）」＋「どれくらいか（効果量・リフト・金額換算）」**。片方だけでは判断できません。

## 7-7. よくある落とし穴

A/B テストは「やった気」になりやすく、罠が多い領域です。

- **覗き見（peeking）**：途中経過を何度も見て「有意になった瞬間」に止める。これは第一種の誤りを激増させる。**事前に期間とサンプルサイズを決め、終わるまで結論を出さない。**
- **p値ハッキング**：セグメントや指標を変えて回し、有意になったものだけ報告する。やってはいけない。
- **多重比較**：たくさん検定すれば、どれかは偶然 p<0.05 になる（20回やれば平均1回）。比較が多いときは補正（Bonferroni 等）や事前の指標を絞る。
- **サンプルサイズ不足**：n が小さいと差を検出できない（検出力不足＝第二種の誤り）。**事前に必要数を見積もる。**
- **「有意でない＝差がない」ではない**：p≥0.05 は「差があるとは言えなかった」だけ。差がないことの証明ではない。
- **新奇性効果・期間の偏り**：施策直後の物珍しさ、曜日・季節の偏りに注意。

> 💡 一番大事な習慣：**テストを始める前に「仮説・主要指標・期間・必要サンプル数・判断基準」を紙に書く。** 後出しの解釈を封じることが、信頼できる結果への近道です。

## 7-8. 意思決定者への報告（結論ファースト）

検定結果はそのままでは伝わりません。モジュール01の「結論→根拠→示唆→提案」で翻訳します。

```
【結論】 新施策 B は CVR を 4.4% → 6.0%（相対 +35%）に改善。偶然ではほぼ説明できない差です（p≈0.0001）。
【根拠】 6,150 vs 6,200 人の A/B テスト。カイ二乗検定で有意（5%水準）。滞在時間も B が有意に長い。
【示唆】 B は購入と滞在の両面で効いている。月10万訪問なら購入が約1,560件/月の増加に相当。
【提案】 B を全面展開。展開後2週間で同等のリフトが続くか（新奇性効果でないか）を継続モニタリング。
```

「p値が0.0001」ではなく **「偶然ではほぼ起きない差」**、効果は **金額・件数** に換算して伝える。これがプロの報告です。

## 7-9. このモジュールのまとめ

- 標本から母集団を語るには、**点ではなく区間（信頼区間）** で考える。
- 検定の手順：**仮説 → 検定統計量 → p値 → α と比較 → 判断**。p値は「H₀ の下で観測以上に極端な差が出る確率」。
- 連続値の差は **t 検定**、割合の差は **カイ二乗 / z 検定**。
- **有意性（p値）と効果量（リフト・金額）はセット** で報告する。
- 覗き見・p値ハッキング・多重比較・サンプル不足の罠を避ける。
