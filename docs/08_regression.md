# 08. 相関と回帰分析

> このモジュールのゴール：**2つ以上の変数の関係を「相関」と「回帰」で読み解き、係数・R² を日本語で正しく解釈できるようになる。**
> 前提：モジュール05（記述統計・分布）とモジュール06（可視化）。相関と因果の話はモジュール01の再確認です。

---

## 8-1. このモジュールでやること

これまでは「1つの変数を眺める（分布・平均）」中心でした。ここからは **「変数Aが変わると変数Bはどう動くか」** を扱います。

- **相関**：2変数が「一緒に動くか」を1つの数字（相関係数）で測る。
- **回帰**：「Aを使ってBを予測・説明する式」を引く。傾き・切片・R² を読む。

題材は、注文データを **顧客ごとに集計したテーブル** です（`status=='completed'` のみを売上とみなす、というモジュール全体の規約に従います）。

| 列 | 意味 |
|----|------|
| `age` | 年齢 |
| `gender` / `acquisition_channel` / `prefecture` | 属性・獲得経路・都道府県 |
| `total_spend` | 累計購入額（円、completed のみ） |
| `n_orders` | 注文回数（completed のみ） |
| `avg_order` | 平均客単価 = total_spend / n_orders |

> 💡 「集計の単位（1行が何を表すか）」を最初に決めるのが回帰の出発点です。
> 今回は **1行 = 1顧客**。売上は注文単位で合計してから顧客単位にまとめます。

## 8-2. 相関 ── 一緒に動くか

### 散布図でまず目で見る
相関係数を計算する前に、必ず **散布図** を描きます。数字だけ見ると、曲線関係や外れ値を見落とすからです。

### ピアソン相関係数
2変数 \(x, y\) の直線的な関係の強さを **−1〜+1** の1つの数字で表すのがピアソンの相関係数 \(r\) です。

\[
r = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum (x_i - \bar{x})^2}\,\sqrt{\sum (y_i - \bar{y})^2}}
\]

- \(r = +1\)：完全に右上がりの直線
- \(r = 0\)：直線的な関係なし
- \(r = -1\)：完全に右下がりの直線

```python
# DataFrame の数値列の相関行列
df[["age", "total_spend", "n_orders", "avg_order"]].corr()

# 2変数だけなら scipy で相関係数と p値の両方
from scipy.stats import pearsonr
r, p = pearsonr(df["n_orders"], df["total_spend"])
```

### 相関の強さの目安（あくまで目安）

| \(|r|\) | ざっくりした言い方 |
|--------|--------------------|
| 0.0 〜 0.2 | ほとんど相関なし |
| 0.2 〜 0.4 | 弱い相関 |
| 0.4 〜 0.7 | 中程度の相関 |
| 0.7 〜 1.0 | 強い相関 |

> ⚠️ この区切りは分野や文脈で変わります。「\(r=0.5\) だから中程度」と機械的に言うより、**散布図と合わせて**判断しましょう。

### 相関 ≠ 因果（モジュール01の再確認）
相関が強くても **「AがBを引き起こす」とは限りません**。

- A→B（原因と結果）
- B→A（逆向き）
- C→A かつ C→B（**第3の変数**。例：アイス販売と水難事故はどちらも「気温」が原因）
- ただの偶然（**見せかけの相関**）

> 💡 相関は「関係がある」までしか言えません。「だから〇〇すれば売上が上がる」は別途、実験（A/Bテスト＝モジュール07）などが必要です。

## 8-3. 単回帰 ── 直線をあてはめる

1つの説明変数 \(x\) で目的変数 \(y\) を説明する直線：

\[
y = a x + b \qquad (a:\text{傾き},\ b:\text{切片})
\]

を、実データに **最もよく合う** ように引きます（残差の二乗和が最小になる直線＝最小二乗法）。

```python
from scipy.stats import linregress
res = linregress(df["n_orders"], df["total_spend"])
res.slope      # 傾き a
res.intercept  # 切片 b
res.rvalue     # 相関係数 r （2乗すると R²）
res.pvalue     # 傾きが0でないかの検定の p値
res.stderr     # 傾きの標準誤差
```

### 傾き・切片の意味
- **傾き \(a\)**：\(x\) が **1増えると** \(y\) が平均 \(a\) だけ変わる。
  例）`n_orders → total_spend` で傾きが約 22,700 なら「注文が1回増えると累計購入額は平均で約2.3万円増える」。
- **切片 \(b\)**：\(x=0\) のときの \(y\) の予測値。今回のように \(x=0\)（注文0回）が現実離れしている場合、切片そのものに深い意味はない（後述の外挿に注意）。

### R²（決定係数）── あてはまりの良さ
\[
R^2 = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y}_i)^2}
\quad(\hat{y}:\text{予測値})
\]

- **\(y\) のばらつきのうち、何割をこの式で説明できたか** を 0〜1 で表す。
- \(R^2 = 0.8\) なら「目的変数のばらつきの約8割を説明できた」。
- **単回帰では \(R^2 = r^2\)**（相関係数の2乗に一致）。

### 残差を必ず確認する
**残差 = 実測値 − 予測値**。横軸に予測値、縦軸に残差をとった **残差プロット** を描き、

- 残差が0の周りに **ランダムに散らばっていれば** あてはまりは妥当。
- 曲がった形・ラッパ状（予測値が大きいほど残差も大きい）が見えたら、直線モデルが不適切のサイン。

> 💡 R² が高くても残差に偏りがあれば要注意。「数字（R²）」と「形（残差プロット）」の両方で判断します。

## 8-4. 重回帰 ── 複数の説明変数

説明変数を複数にすると：

\[
y = b + a_1 x_1 + a_2 x_2 + \cdots + a_k x_k
\]

`sklearn` は使わず、**numpy の最小二乗 `np.linalg.lstsq`** で実装します。切片を入れるため **定数項（全部1の列）を足す** のがポイントです。

```python
import numpy as np

X = df[["age", "n_orders"]].to_numpy(float)
y = df["total_spend"].to_numpy(float)

# 先頭に「全部1」の列を足す → 切片に対応する係数が得られる
Xc = np.column_stack([np.ones(len(X)), X])
beta, *_ = np.linalg.lstsq(Xc, y, rcond=None)
# beta[0]=切片, beta[1]=ageの係数, beta[2]=n_ordersの係数

y_hat = Xc @ beta                 # 予測値
ss_res = ((y - y_hat) ** 2).sum() # 残差平方和
ss_tot = ((y - y.mean()) ** 2).sum()
r2 = 1 - ss_res / ss_tot          # 決定係数 R²（手計算）
```

### 係数の解釈 ── 「他を一定としたとき」
重回帰の係数 \(a_j\) は、**他の説明変数を固定したまま** \(x_j\) を1増やしたときの \(y\) の平均変化量です。

- 「`n_orders` の係数 22,700」＝**年齢など他を同じにして**、注文回数が1増えると累計購入額は平均で約2.3万円増える。
- 単回帰の傾きと重回帰の係数は、他の変数との関係しだいで **値が変わる**（同じとは限らない）。

> 💡 「他を一定としたとき（holding others constant）」が重回帰の係数解釈の決まり文句。
> ここを省くと「年齢が効いている」のような誤読を招きます。

## 8-5. カテゴリ変数の扱い ── ダミー変数化

`acquisition_channel`（検索広告/SNS広告/…）のような **文字のカテゴリ** はそのまま回帰に入れられません。
**ダミー変数**（0/1の列）に展開します。

```python
dum = pd.get_dummies(df["acquisition_channel"], prefix="ch", drop_first=True).astype(float)
```

- `drop_first=True` で1カテゴリを **基準** として落とす（**ダミー変数の罠**＝多重共線性の回避）。
- 残ったダミーの係数は **「基準カテゴリと比べた差」** と読む。
  例）基準が「SNS広告」で `ch_メルマガ` の係数が −2,000 なら「SNS広告に比べてメルマガ経由は平均2,000円少ない（他を一定として）」。

> 💡 ダミーを足すと R² は基本的に下がりません（説明変数が増えるため）。
> だから **「R² が上がった＝良いモデル」とは限らない**（過学習のさわり、次節）。

## 8-6. 解釈の注意 ── ここを外すと事故る

- **外挿の危険**：データの範囲外を予測しない。注文0〜27回で引いた式で「注文100回なら？」は保証外。
- **過学習のさわり**：説明変数を増やすほど学習データへのあてはまり（R²）は上がるが、未知データには弱くなることがある。「R² が高い＝良い」ではない。
- **相関と因果**：回帰の係数は **関係の強さ** であって、**因果**を意味しない。「係数が正だから増やせば売上が上がる」とは言えない。
- **見せかけの回帰**：時系列など、本来無関係なもの同士でも、共通のトレンドがあると高い R² が出てしまうことがある。
- **欠損・スケール**：欠損行は落とすか埋めるか方針を決める。単位の違う変数（年齢 vs 円）は、係数の **大きさの直接比較ができない** ことに注意。

### 予測のための回帰 vs 説明のための回帰
同じ回帰でも目的が2つあります。

| 目的 | 重視すること |
|------|--------------|
| **予測** | 未知データを当てる精度（R²、未知データでの誤差） |
| **説明** | 各変数が結果にどう効くか（係数の符号・大きさ・解釈） |

本講座は主に **説明のための回帰**（係数を日本語で解釈する）を扱います。
「この変数を1増やすと結果がどう動くか」を、因果に踏み込みすぎずに語れることがゴールです。

> 🧭 合言葉：**「この回帰から言えること／言えないこと」を1文ずつ書く。**
> 言えること＝関係の向きと強さ。言えないこと＝因果、範囲外の予測。

---

## 8-7. まとめ

- 相関係数 \(r\) は2変数の **直線関係の強さ**（−1〜+1）。必ず散布図とセットで見る。
- 単回帰：`linregress` で傾き・切片・R²。傾きは「\(x\) が1増えたときの \(y\) の平均変化」。
- 重回帰：`np.linalg.lstsq`（定数項列を足す）。係数は **「他を一定としたとき」** の効果。R² は手計算で。
- カテゴリは `pd.get_dummies(..., drop_first=True)` でダミー化。係数は基準との差。
- 相関も回帰も **因果ではない**。外挿・過学習・見せかけに注意。
